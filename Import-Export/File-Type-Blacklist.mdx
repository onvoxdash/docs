Control which file types people are blocked from uploading to your organization by
managing a blacklist of file extensions. You can export the current list and import a
replacement. Found in your organization's file-type settings.

<Info>
  This is a paid feature (Customize File-Type Blacklist) and requires Editor access to
  the organization.
</Info>

## How to import

<Steps>
  <Step title="List the extensions">
    Put one file extension per row under the **Extensions** column. A leading dot is
    optional — `exe` and `.exe` are treated the same.
  </Step>
  <Step title="Upload">
    Import the file. The uploaded list **replaces** the organization's current blacklist.
  </Step>
</Steps>

## Columns

| Column | Required | Description | Example |
| --- | --- | --- | --- |
| Extensions | Yes | One file extension per row. | `.exe` |

## Limits & rules

| Rule | Value |
| --- | --- |
| File type | `.xlsx` only (contents verified) |
| Maximum size | 2 MB |
| Maximum extensions | 2,000 |
| Valid extension | letters/numbers only, up to 31 characters (e.g. `exe`, `bat`, `7z`) |

<Warning>
  Importing **replaces** the whole list. A file that contains no valid extensions is
  rejected so it can't accidentally wipe your blacklist. Invalid entries (symbols,
  spaces, compound names like `tar.gz`) are skipped.
</Warning>

## Troubleshooting

| Problem | What it means |
| --- | --- |
| Invalid file | Not a real `.xlsx`, empty, too large, or it produced no valid extensions. |
| Invalid organization access | You lack Editor access, or your plan doesn't include this feature. |
