---
title: "How to Add a Map in VoxDash"
---

VoxDash includes a library of geographic maps that can be used in dashboards, reports, and data visualizations. If the map you need is not already available, you can add a custom map through the **Data Provider** panel.

VoxDash supports maps created from **GeoJSON** files. You can obtain GeoJSON data from mapping resources such as Highcharts Maps and then import it into VoxDash for use in your projects.

## Before You Begin

If the map you need is not available in VoxDash:

1. Visit the [Highcharts Maps website - GeoJSON](/https://www.highcharts.com/docs/maps/custom-geojson-maps) and [Highcharts Maps website - Custom Map](/https://www.highcharts.com/docs/maps/create-custom-maps).
2. Search for the country, region, province, state, district, or custom geographic area you want to use.
3. Download or copy the corresponding **GeoJSON** file.
4. Use the GeoJSON data when creating a new map in VoxDash.
5. Use the using [Highcharts online map converter tool](/https://highcharts.github.io/map-from-svg/) to construct an interactive map with data values. 

> **Note:** VoxDash uses GeoJSON data to define geographic boundaries and regions. Ensure that your GeoJSON file is valid and properly formatted before importing it.

## Steps to Add a New Map

### Step 1: Navigate to the Maps Page and Create New Map

Open the **Data Provider** panel and navigate to the **Maps** section.

The Maps page displays all maps currently available in your VoxDash environment and allows administrators to create, edit, and manage custom maps.

On the Maps page, click the **Add Map** button.

This opens the **Create Map** form, where you can define the map details and upload the geographic data.

### Step 3: Select the Appropriate Region

Choose the geographic region that corresponds to your map.

You can either:

* Search for an existing region using the search box.
* Select a region from the available list.

Assigning the correct region ensures that:

* The map is categorized properly.
* Geographic filtering works correctly.
* Users can navigate through geographic hierarchies in dashboards and reports.

For example, a map showing counties in Kenya should be associated with the **Kenya** region.

### Step 4: Add the GeoJSON Data

In the **GeoJSON** field, paste the GeoJSON code for your map.

  <img
    src="/images/map-json.png"
    alt="add parent"
    style={{ borderRadius:"0.5rem" }}
  />
</Frame>

GeoJSON is a standard format used to represent geographic features such as:

* Countries
* States or provinces
* Counties
* Districts
* Cities
* Custom geographic boundaries

The GeoJSON file contains both the geographic shapes and any associated properties required to render the map correctly.

#### Tips

* Verify that the GeoJSON is valid before saving.
* Ensure that region names within the GeoJSON match your intended geography.
* Large or complex GeoJSON files may take longer to process.

> **Important:** Invalid or malformed GeoJSON data may prevent the map from displaying correctly in VoxDash.

### Step 5: Add a New Region (If Required)

If the geography you want to use does not exist in the region list, you must first create a new region.

Click **Add New Region** and complete the required fields.

#### Define the Region Name

Enter a clear and accurate name for the region.

Examples:

* Nairobi County
* Westlands District
* Central Province
* Greater Accra Region

The name should reflect the actual geographic area represented by the map.

#### Select the Parent Region

Every region in VoxDash belongs to a geographic hierarchy.

Selecting a parent region ensures that the new region is connected correctly to the existing geographic structure.

### Understanding Geographic Hierarchies

Regions are typically organized from larger areas to smaller areas.

For example:

| Level                 | Example            |
| --------------------- | ------------------ |
| Continent             | Africa             |
| Country               | Kenya              |
| Province / State      | Nairobi County     |
| District / Sub-region | Westlands District |

This hierarchy allows VoxDash to:

* Aggregate data from lower levels to higher levels.
* Filter dashboards by geographic region.
* Support drill-down analysis.
* Display geographic relationships consistently across reports.

#### Example

If you are adding **Westlands District**, the hierarchy may be:

```
Africa
└── Kenya
    └── Nairobi County
        └── Westlands District
```

When users analyze data, they can drill down from Kenya → Nairobi County → Westlands District.

## Add Additional Parent Regions (Optional)

Some geographic entities may belong to multiple parent regions or classifications.

If applicable, add all relevant parent regions to ensure that the geography is correctly connected within the VoxDash hierarchy.

This improves data filtering and reporting flexibility.

### Add a Wikipedia Link (Optional)

If a Wikipedia page exists for the region, you may provide the URL in the **Wikipedia Link** field.

Adding a Wikipedia link gives users quick access to additional information about the geography, including:

* Demographics
* Administrative structure
* Historical background
* Geographic details

# Step 6: Create the Map

After completing all required fields:

1. Review the map name, region selection, and GeoJSON data.
2. Confirm that the region hierarchy is configured correctly.
3. Click **Create** in the lower-right corner of the page.

VoxDash will save the map and make it available for use in dashboards, reports, visualizations, and geographic analysis.

## Verifying the Map

After creating the map:

1. Return to the **Maps** page.
2. Locate the newly created map in the list.
3. Open the map to confirm that:

   * The boundaries render correctly.
   * All geographic regions appear as expected.
   * Region names are displayed properly.
   * The map is associated with the correct geographic hierarchy.

If any issues are detected, edit the map and verify the GeoJSON data and region configuration.
