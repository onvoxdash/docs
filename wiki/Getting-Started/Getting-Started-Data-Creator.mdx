Who this is for

Use this guide if you prepare and manage data — creating projects, importing files (including SPSS), defining structure, and controlling access for your organization.

Quick checklist

- Confirm plan and add‑ons: [SaaS Plans & Usage](/wiki/Billing,-Subscriptions-&-System-Usage/SaaS-Usage-&-Billing-Summary), [Purchase a Plan](/wiki/Billing,-Subscriptions-&-System-Usage/Purchase-Subscription-Plan), [Purchase an Add‑on](/wiki/Billing,-Subscriptions-&-System-Usage/Buy-Add--Ons)
- Plan your project structure: [Projects List](/wiki/Data-Projects/Data-Project-List), [Project Page](/wiki/Data-Projects/Data-Project-Overrview)
- Prepare your files: [Sample File](/wiki/Data-Management/Add-Sample-File), [Placeholder File](/wiki/Data-Management/Add-Placeholder-File), [SPSS Overview](/wiki/SPSS-&-Data-Entry/Working-with-SPSS-Files)
- Import data: [Data Entry](/wiki/data-entry), [Bulk Upload](/wiki/Data-Management/Bulk-File-Upload), [Multi Bulk Upload](/wiki/Data-Management/Multi-Folder-Upload), [SPSS Data Entry](/wiki/SPSS-&-Data-Entry/SPSS-Data-Entry-Guide)
- Validate and document: [Data Profile](/wiki/SPSS-&-Data-Entry/Data-Profile-Guide)
- Configure access: [User Management](/wiki/user-management), [User Groups](/wiki/user-groups), [Entity Access](/wiki/entity-access), [Security Settings](/wiki/security-settings), [Public Access](/wiki/public-access)
- Optional entities: vendors & individuals — [Vendor List](/wiki/vendor-list), [Vendor Page](/wiki/vendor-page), [Create Vendor](/wiki/create-vendor-for-organization), [Choose Vendor from Header](/wiki/choose-vendor-from-header), [Vendor Individual Roles](/wiki/vendor-individual-roles)
- Visualize & launch: [Maps](/wiki/map), [Drilldown Map](/wiki/drilldown-map), [Map Visualization (Questions)](/wiki/map-visualization-question-page), [Homepage Setup](/wiki/homepage-setup)

1. Set up billing and capacity

- Review your entitlement and usage: [SaaS Plans & Usage](/wiki/Billing,-Subscriptions-&-System-Usage/SaaS-Usage-&-Billing-Summary)
- Activate a plan and required add‑ons: [Purchase a Plan](/wiki/Billing,-Subscriptions-&-System-Usage/Purchase-Subscription-Plan), [Purchase an Add‑on](/wiki/Billing,-Subscriptions-&-System-Usage/Buy-Add--Ons)

2. Create and structure a project

- From the [Projects List](/wiki/data-project-list), create a new project and open its [Project Page](/wiki/data-project-page)
- Decide entities you’ll manage (vendors, individuals) and the required fields

3. Prepare and import your data

- Start with a [Sample File](/wiki/sample-file) to align columns and types
- For SPSS workflows, see [SPSS Overview](/wiki/spss), import via [SPSS Data Entry](/wiki/spss-data-entry), and align fields in [Map SPSS Fields](/wiki/setup-map-spss)
- Import new records with [Data Entry](/wiki/data-entry), [Bulk Upload](/wiki/bulk-upload), or [Multi Bulk Upload](/wiki/multi-bulk-upload)

4. Validate structure and documentation

- Open the [Data Profile](/wiki/data-profile) to review and document fields, constraints, and metadata
- Iterate imports until validation passes

5. Configure users and governance

- Add users and assign them to groups: [User Management](/wiki/user-management), [User Groups](/wiki/user-groups)
- Scope access by entity or project: [Entity Access](/wiki/entity-access)
- Enforce auth controls and policies: [Security Settings](/wiki/security-settings)
- If appropriate, enable public‑read content: [Public Access](/wiki/public-access)

6. Optional: model vendors and individuals

- Manage organizations: [Vendor List](/wiki/vendor-list), [Vendor Page](/wiki/vendor-page), [Create Vendor](/wiki/create-vendor-for-organization), [Choose Vendor from Header](/wiki/choose-vendor-from-header)
- Set roles and responsibilities: [Vendor Individual Roles](/wiki/vendor-individual-roles)

7. Visualize and launch

- Build maps and geospatial drilldowns: [Maps](/wiki/map), [Drilldown Map](/wiki/drilldown-map)
- Visualize questions on maps: [Map Visualization (Questions)](/wiki/map-visualization-question-page)
- Curate the landing experience: [Homepage Setup](/wiki/homepage-setup)

What’s next

- Measure adoption and key metrics in [Dashboard](/wiki/dashboard)
- Help colleagues get set up using the companion guide: [Getting Started: Data User](/wiki/getting-started-data-user)
