title: "Field Value"
---

The **Field Value** page allows users to track the **approval status** of requests submitted across different sections of the platform. This page provides visibility into the review process and helps users monitor whether submitted items have been approved, rejected, or require further action.

The page is organized into several tabs, each representing a different category of submitted requests.

## Topics Tab

The **Topics** tab displays all topic-related requests submitted by the user. It provides an overview of each topic along with its current approval status and feedback from the administrator.

The table includes the following information:

| Column              | Description                                                                                      |
| ------------------- | ------------------------------------------------------------------------------------------------ |
| **Topics**          | The name or identifier of the submitted topic.                                                   |
| **Name**            | The associated name or title of the topic entry.                                                 |
| **Creation Date**   | The date when the topic was originally created and submitted for review.                         |
| **Approved Status** | Indicates the current review status, such as *Pending*, *Approved*, or *Rejected*.               |
| **Admin Message**   | Feedback, comments, or additional notes provided by the administrator during the review process. |

Use this tab to monitor the review progress of submitted topics and check administrator feedback when updates or corrections are required.

## Data Entry Items Tab

The **Data Entry Items** tab displays all submitted data entry fields that are currently under review or have already been processed.

This section helps users track the approval progress of individual data entry fields and review administrator comments when necessary.

The table includes the following columns:

| Column              | Description                                                                                       |
| ------------------- | ------------------------------------------------------------------------------------------------- |
| **Name**            | The name of the submitted data entry item.                                                        |
| **Field Type**      | The type of field created, such as text, numeric, date, dropdown, or other supported field types. |
| **Creation Date**   | The date when the field was created and submitted for approval.                                   |
| **Approved Status** | The current approval state of the submitted item.                                                 |
| **Admin Message**   | Any comments, feedback, or requested changes provided by the administrator.                       |

<Frame>
  <img src="/images/field-entry.png" alt="Field Value Roles" style={{ borderRadius: '0.5rem' }} />
</Frame>

This tab is useful for verifying whether newly created data fields have been approved and identifying any required modifications.

## Data Roles Tab

The **Data Roles** tab allows users to review the approval history and status of submitted data roles.

This section helps users understand who reviewed the request and when the latest status update was made.

The table includes the following information:

| Column                      | Description                                                                       |
| --------------------------- | --------------------------------------------------------------------------------- |
| **Title**                   | The name or title of the submitted data role.                                     |
| **Approved By**             | The administrator or reviewer responsible for approving or rejecting the request. |
| **Admin Message**           | Comments or feedback left by the administrator regarding the submission.          |
| **Last Change Status Date** | The date when the approval status was last updated.                               |
| **Approved Status**         | The current approval status of the data role request.                             |

<Frame>
  <img src="/images/field-role.png" alt="Field Value Roles" style={{ borderRadius: '0.5rem' }} />
</Frame>

Use this tab to track the approval workflow for role-related requests and review administrator feedback.

## Geographic Region Tab

The **Geographic Region** tab displays all geographic region requests submitted for approval.

This section helps users track region-related submissions and review approval decisions made by administrators.

The table contains the following columns:

| Column                      | Description                                                              |
| --------------------------- | ------------------------------------------------------------------------ |
| **Name**                    | The name of the geographic region submitted.                             |
| **Logo**                    | The logo or visual identifier associated with the region, if applicable. |
| **Creation Date**           | The date when the region was created and submitted for review.           |
| **Admin Message**           | Feedback or comments provided by the administrator during review.        |
| **Last Change Status Date** | The most recent date when the approval status changed.                   |
| **Approved Status**         | Indicates whether the request is *Pending*, *Approved*, or *Rejected*.   |

<Frame>
  <img src="/images/field-geo.png" alt="Field Value Geographic" style={{ borderRadius: '0.5rem' }} />
</Frame>

Use this tab to monitor the approval process for geographic regions and check for any administrator feedback that may require further action.

### General Note

Each section on the **Field Value** page helps users keep track of submitted requests and stay informed about their approval progress. If an item is rejected or requires changes, review the **Admin Message** column for additional details and take the necessary corrective action before resubmitting.
