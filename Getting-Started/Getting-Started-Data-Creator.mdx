Who this is for

Use this guide if you prepare and manage data — creating projects, importing files (including SPSS), defining structure, and controlling access for your organization.

Quick checklist

- Confirm plan and add‑ons: [SaaS Plans & Usage](/Billing,-Subscriptions-&-System-Usage/SaaS-Usage-&-Billing-Summary), [Purchase a Plan](/Billing,-Subscriptions-&-System-Usage/Purchase-Subscription-Plan), [Purchase an Add‑on](/Billing,-Subscriptions-&-System-Usage/Buy-Add--Ons)
- Plan your project structure: [Projects List](/Data-Projects/Data-Project-List), [Project Page](/Data-Projects/Data-Project-Overrview)
- Prepare your files: [Sample File](/Data-Management/Add-Sample-File), [Placeholder File](/Data-Management/Add-Placeholder-File), [SPSS Overview](/SPSS-&-Data-Entry/Working-with-SPSS-Files)
- Import data: [Data Entry](/data-entry), [Bulk Upload](/Data-Management/Bulk-File-Upload), [Multi Bulk Upload](/Data-Management/Multi-Folder-Upload), [SPSS Data Entry](/SPSS-&-Data-Entry/SPSS-Data-Entry-Guide)
- Validate and document: [Data Profile](/SPSS-&-Data-Entry/Data-Profile-Guide)
- Configure access: [User Management](/user-management), [User Groups](/user-groups), [Entity Access](/entity-access), [Security Settings](/security-settings), [Public Access](/public-access)
- Optional entities: vendors & individuals — [Vendor List](/vendor-list), [Vendor Page](/vendor-page), [Create Vendor](/create-vendor-for-organization), [Choose Vendor from Header](/choose-vendor-from-header), [Vendor Individual Roles](/vendor-individual-roles)
- Visualize & launch: [Maps](/map), [Drilldown Map](/drilldown-map), [Map Visualization (Questions)](/map-visualization-question-page), [Homepage Setup](/homepage-setup)

1. Set up billing and capacity

- Review your entitlement and usage: [SaaS Plans & Usage](/Billing,-Subscriptions-&-System-Usage/SaaS-Usage-&-Billing-Summary)
- Activate a plan and required add‑ons: [Purchase a Plan](/Billing,-Subscriptions-&-System-Usage/Purchase-Subscription-Plan), [Purchase an Add‑on](/Billing,-Subscriptions-&-System-Usage/Buy-Add--Ons)

2. Create and structure a project

- From the [Projects List](/data-project-list), create a new project and open its [Project Page](/data-project-page)
- Decide entities you’ll manage (vendors, individuals) and the required fields

3. Prepare and import your data

- Start with a [Sample File](/sample-file) to align columns and types
- For SPSS workflows, see [SPSS Overview](/spss), import via [SPSS Data Entry](/spss-data-entry), and align fields in [Map SPSS Fields](/setup-map-spss)
- Import new records with [Data Entry](/data-entry), [Bulk Upload](/bulk-upload), or [Multi Bulk Upload](/multi-bulk-upload)

4. Validate structure and documentation

- Open the [Data Profile](/data-profile) to review and document fields, constraints, and metadata
- Iterate imports until validation passes

5. Configure users and governance

- Add users and assign them to groups: [User Management](/user-management), [User Groups](/user-groups)
- Scope access by entity or project: [Entity Access](/entity-access)
- Enforce auth controls and policies: [Security Settings](/security-settings)
- If appropriate, enable public‑read content: [Public Access](/public-access)

6. Optional: model vendors and individuals

- Manage organizations: [Vendor List](/vendor-list), [Vendor Page](/vendor-page), [Create Vendor](/create-vendor-for-organization), [Choose Vendor from Header](/choose-vendor-from-header)
- Set roles and responsibilities: [Vendor Individual Roles](/vendor-individual-roles)

7. Visualize and launch

- Build maps and geospatial drilldowns: [Maps](/map), [Drilldown Map](/drilldown-map)
- Visualize questions on maps: [Map Visualization (Questions)](/map-visualization-question-page)
- Curate the landing experience: [Homepage Setup](/homepage-setup)

What’s next
