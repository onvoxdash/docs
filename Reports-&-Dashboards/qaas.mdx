---
title: "QaaS (Questions‑as‑a‑Service) Reports"
---

The **QaaS category** in VoxDash provides specialized analytics for survey‑based datasets. These reports help teams understand how users engage with survey questions, what analyses they perform, and how frequently cross‑tabulation tools are used.

This section explains each report type in detail.

## Question Views Count

This report returns the **total number of page views for individual survey questions** within a selected timeframe.

What it measures:
- Total views per question
- Engagement trends across surveys or datasets
- Popularity of specific questions among users

Typical use cases:
- Identifying high‑traffic questions
- Evaluating interest in certain topics or survey sections
- Supporting reporting for research or customer‐insight teams

## Cross‑Tabulation Count

This summary report shows the **total number of cross‑tabulation analyses run by users**.

What it helps you understand:
- How frequently users perform comparative analysis
- Engagement with analytical tools inside VoxDash
- Overall usage patterns for the cross‑tab feature

Use cases:
- Monitoring the adoption of advanced analytics features
- Measuring user activity for enterprise usage reports
- Planning resource allocation or performance tuning

## Question Page Views

This detailed log tracks every view of a **survey question page**, capturing all contextual metadata around the event.

Information included:
- Question title  
- Survey name  
- Associated dataset  
- Viewer name/email (if available)  
- Indicator showing whether filters were applied  
- View type (e.g., direct view, navigation, deep link)

Why it’s useful:
- Understand how users navigate surveys  
- Analyze which questions attract the most attention  
- Detect usage patterns such as filtered vs. unfiltered exploration  
- Identify potential usability issues in long surveys

Common use cases:
- UX research on survey interaction
- Evaluating user interest in specific question types
- Supporting audit and compliance logs

## Cross‑Tabulation Analyses

This is a detailed operational log of all **cross‑tabulation analyses** performed by users.  
Each entry gives insight into how users compare two different survey questions.

Information included:
- Main (primary) question  
- Secondary/comparison question  
- Survey and dataset names  
- Organization or workspace  
- User details (name/email)  
- Filter status (whether filters were applied)  
- Analysis type (e.g., standard cross‑tab, segmented view, advanced filters)

Why this matters:
- Understand which comparisons users find most valuable  
- Track how decision‑makers analyze survey results  
- Identify trends in analytical behavior across teams  
- Support audit trails for regulated environments  

Use cases:
- Enhancing survey design based on analysis behavior  
- Usage‑based billing or enterprise analytics reporting  
- Detecting heavy usage for performance scaling
