---
title: "Search Engine Visibility Reports"
---


The **Search Engine** report category in VoxDash helps data providers understand which content from their workspace is publicly discoverable through external search engines such as Google, Bing, or other web crawlers.  

These reports are especially useful for:
- Monitoring public visibility  
- Ensuring compliance with data‑sharing policies  
- Tracking SEO exposure  
- Auditing content accessibility

Below is a detailed explanation of each report in this category.

## Search Engine Indexed Files

This report lists all **downloadable files** that have been discovered and indexed by search engine crawlers.

Information included:
- File name  
- Entity type (e.g., dataset file, attachment, resource file)  
- Last modified by (user name/email)  
- Date the file was indexed by the search engine  

Why it matters:
- Helps verify which files are publicly visible  
- Useful for SEO optimization for public datasets  
- Supports compliance reviews by confirming allowed public exposure  

Common use cases:
- Auditing public file accessibility  
- Monitoring which documents search engines have cached  
- Detecting unexpected public exposure caused by misconfigured permissions 

## Search Engine Indexed Questions

This report lists all **survey questions** that have been crawled and indexed by search engines.

Information included:
- Question title  
- Associated survey (if relevant)  
- Index date  

Why this is useful:
- Allows you to track public visibility of survey questions  
- Helps ensure that only intended questions are discoverable  
- Supports SEO and content engagement analysis 

## Search Engine Indexed Datasets

This report lists all **datasets** that have been detected and indexed by search engines.

Information included:
- Dataset name  
- Index date  

Why this matters:
- Allows dataset owners to track SEO visibility  
- Helps teams ensure the right datasets are discoverable  
- Useful for auditing privacy settings and public access configurations 

## Search Engine Indexed Vendors

This report lists all **vendor profile pages** that have been indexed by search engines.

Information included:
- Vendor name  
- Index date  

Why it’s helpful:
- Ensures vendor pages intended for public visibility are being indexed  
- Supports vendor growth by improving search discoverability  
- Useful for partner reporting and SEO tracking

## Search Engine Indexed Individuals

This report lists all **individual/person profile pages** that have been indexed by search engines.

Information included:
- Person name  
- Index date  

Why this report is useful:
- Helps maintain transparency regarding profile visibility  
- Ensures compliance with privacy preferences  
- Useful for personal branding analysis for contributors or researchers
